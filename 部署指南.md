# OJ 判题平台 - 从零部署指南

## 前置条件：一台 Windows 电脑，啥也没装

---

## 第一步：安装 Python 3

1. 打开浏览器，访问 https://www.python.org/downloads/
2. 点击黄色大按钮下载 Python 3.12（或更高版本）
3. 运行安装程序，**务必勾选 "Add Python to PATH"**（底部复选框）
4. 点 Install Now，等待完成
5. 验证：按 `Win+R`，输入 `cmd`，回车，输入：
   ```
   python --version
   ```
   应该显示 `Python 3.12.x`

---

## 第二步：安装 MySQL 8.0

1. 访问 https://dev.mysql.com/downloads/installer/
2. 下载 `mysql-installer-community-8.0.x.msi`（约 400MB）
3. 运行安装程序，选择 "Developer Default"
4. 一路 Next，到 Accounts and Roles 这一步时：
   - **MySQL Root Password：zjy200646**
   - **Confirm Password：zjy200646**
   - 记牢这个密码，项目配置用的就是这个
5. 继续 Next → Execute → Finish
6. 验证：开始菜单找到 MySQL 8.0 Command Line Client，输入密码 `zjy200646` 能进去就行

---

## 第三步：安装 C++ 编译器（MinGW-w64）

1. 访问 https://winlibs.com/
2. 下载 GCC 对应你系统的版本（推荐 `GCC 14.x + MinGW-w64 + MSVCRT`，选 `x86_64-posix-seh`）
3. 解压到 `C:\mingw64`（直接放C盘根目录，路径别带空格）
4. 配置环境变量：
   - 按 `Win+R`，输入 `sysdm.cpl`，回车
   - 点 "高级" → "环境变量"
   - 在 "系统变量" 里找到 `Path`，双击
   - 点 "新建"，输入 `C:\mingw64\bin`
   - 确定保存
5. 验证：新开一个 cmd 窗口，输入：
   ```
   g++ --version
   ```
   应该显示版本号

## 第四步：部署项目

### 4.1 解压项目

把收到的压缩包解压，假设解压到 `E:\oj_project`
最终结构应该是：
```
E:\oj_project\
  app.py
  config.py
  requirements.txt
  db\
    schema.sql
  judge\
    judge.cpp
    judge.exe   ← 如果有的话
  templates\
    ...
  static\
    uploads\
```

### 4.2 安装 Python 依赖

打开 cmd：
```
cd /d E:\oj_project
pip install -r requirements.txt
```

### 4.3 创建数据库

```
mysql -u root -pzjy200646 < db\schema.sql
```

如果报错 "mysql 不是内部命令"，到 `C:\Program Files\MySQL\MySQL Server 8.0\bin\` 目录下执行，或把该路径加入环境变量 Path。

### 4.4 编译判题程序（如果 judge.exe 不存在）

```
cd judge
g++ -o judge.exe judge.cpp -static -O2 -lpsapi
```

如果 `judge.exe` 已经存在（别人已经编译好），跳过这步。

### 4.5 启动！

```
cd /d E:\oj_project
python app.py
```

看到类似输出：
```
 * Running on http://127.0.0.1:5000
```
表示启动成功。

## 常见问题

**Q: pip install 报错**
A: 检查 Python 是否装好（`python --version`），检查网络是否正常。

**Q: mysql 导入报错 "Access denied"**
A: 确认安装 MySQL 时 Root 密码设为 `zjy200646`。如果设了其他密码，修改 `config.py` 中 `DB_PASSWORD` 为你设的密码。

**Q: g++ 编译报错 `-lpsapi` 找不到**
A: MinGW 版本问题，去掉 `-lpsapi` 参数再试：
```
g++ -o judge.exe judge.cpp -static -O2
```
（psapi 库在某些 MinGW 发行版已内置）

**Q: 启动后页面报 500 错误**
A: 肯定是数据库没连上。检查：
1. MySQL 服务是否启动（任务管理器 → 服务 → MySQL80 是否"正在运行"）
2. `config.py` 里的密码对不对
3. 数据库 `oj_platform` 是否存在（`mysql -u root -pzjy200646 -e "SHOW DATABASES;"`）

**Q: 提交代码后一直 Pending / System Error**
A: judge.exe 没编译成功或不在 judge 目录下。确认 `judge\judge.exe` 存在，且在项目根目录运行 `python app.py`（不要在 judge 子目录里运行）。

**Q: 端口 5000 被占用**
A: 改 `app.py` 最后一行 `port=5000` 为其他端口（如 `port=8080`）。
